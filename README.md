# Словник — ქართული

Локальний Flask-сервер для запису грузинських слів з телефону, поки вчу мову на LingWing.

## Запуск

```
pip install -r requirements.txt
python app.py
```

Або подвійний клік на `start.bat`.

Консоль покаже дві адреси (`https://`, якщо на комп'ютері встановлено mkcert — див. нижче, інакше `http://`):
- `127.0.0.1:5000` — з цього комп'ютера
- `<local-ip>:5000` — з телефону (той самий Wi-Fi). IP може змінюватись після перезавантаження роутера — дивись актуальний в консолі при кожному старті; сертифікат для нової IP перегенерується автоматично.

З телефону відкрий адресу в браузері і додай на головний екран ("Add to Home Screen") — буде як окрема аплікація.

При першому запуску Windows Firewall може спитати дозвіл для Python — треба дозволити для приватних мереж, інакше телефон не з'єднається.

## Тести

```
python tests/run_all.py
```

Сервер має бути запущений (`python app.py`) — тести звертаються до нього по HTTP. Прогонить усі серверні тести (`tests/test_*.py`) і клієнтські (`tests/js/test_*.js`, потрібен Node.js — перевіряють JS-логіку прямо з `templates/index.html` через ізольований `eval`).

- Усі тести створюють лише власні тестові слова (з префіксом на кшталт `test-api-`, `test-srs-`, `test-imp-`) і прибирають їх за собою в кінці — безпечно ганяти проти реального словника, справжні слова не зачіпаються.
- `tests/test_chat_api.py` перевіряє лише безкоштовні шляхи (конфігурація, валідація) і **ніколи не надсилає справжніх повідомлень до Claude** — щоб прогін тестів не витрачав платні токени.
- `run_all.py` повертає ненульовий код виходу, якщо щось провалилось — зручно перед комітом чи в CI.

## Що вміє

- Додавання слова: грузинська, переклад, приклад/нотатка (необов'язково)
- Пошук по грузинській і перекладу (миттєвий, працює й офлайн)
- Редагування та видалення
- Експорт у CSV (кнопка "⬇ CSV") — щоб потім віддати AI на перевірку правильності перекладів/прикладів. Генерується прямо в браузері з локальних даних (Blob + `download`), працює офлайн і не "виносить" зі standalone-PWA на iOS, як зробило б звичайне посилання на серверний маршрут. Містить колонку `uuid` — вона потрібна для імпорту, проси AI її не чіпати
- **Імпорт CSV назад** (кнопка "⬆ CSV") — замикає цикл AI-перевірки: експортував → AI виправив → імпортував. Рядок з відомим `uuid` оновлює наявне слово (SRS-прогрес зберігається), з порожнім/невідомим — створює нове. Захисна сітка від поламаних AI uuid: якщо uuid не знайдено, але в базі рівно одне слово з таким самим грузинським написанням — оновлюється воно, а не створюється дублікат (при кількох омонімах сітка не вгадує і створює нове). Перед застосуванням показується підсумок, повторний імпорт того ж файлу ідемпотентний. Колонки розпізнаються по заголовку, порядок неважливий; обов'язкові лише `georgian` і `translation`. Працює тільки онлайн
- **Теги/категорії** — необов'язкове поле "теги через кому" при додаванні й редагуванні ("їжа", "дієслова", "урок 12"...). Теги нормалізуються (нижній регістр, без дублів). Над списком з'являється панель чипів з лічильниками: тап фільтрує список, повторний тап знімає фільтр. Активний тег обмежує і **повторення** — можна тренувати одну тему окремо. Теги входять у CSV-експорт/імпорт, тож AI може категоризувати словник за тебе
- **Масове перейменування тега** — коли обраний конкретний тег (фільтр активний), поруч із чипами з'являється кнопка "✎": перейменовує тег у всіх словах одразу (зручно виправити одруківку в тезі, коли слів уже багато). Порівняння точне по токену, не по підрядку (перейменування "їжа" не зачепить "їжа2"); якщо слово вже мало цільовий тег — дублікат не з'являється. Потребує з'єднання з сервером
- **Слово дня** — картка над формою додавання: детерміновано випадкове слово зі словника, змінюється раз на добу. Переклад прихований до тапу (маленька вправа на пригадування замість пасивного показу); після тапу лишається відкритим до кінця дня (`localStorage`)
- **Режим друкування відповіді в повтореннях** (типово увімкнений; кнопка "⌨️/👁" в сесії перемикає на класичний показ, вибір запам'ятовується). Друкуєш відповідь → "Перевірити" → вердикт ✅/❌ + правильна відповідь, фінальну оцінку однаково даєш сам (раптом синонім). Перевірка поблажлива: ігнорує регістр/розділові/дужки, зараховує будь-який варіант через кому ("дякую" ≈ "дякую, спасибі"), прощає одну одруківку словам від 5 літер. Порожній ввід + "Перевірити" = просто показати відповідь без вердикту
- **Строгий режим** (кнопка "🔒/🔓" поруч із ⌨️/👁, видима лише в режимі друкування; вибір запам'ятовується) — вимикає прощення одруківок, щоб точно вивчити написання буква в букву (регістр/розділові/дужки й варіанти через кому досі ігноруються — це про орфографію, не про формат)
- **Клавіатурне керування повтореннями** (для десктопа): Enter — перевірити/показати відповідь; на екрані оцінки `1` — "не знав", `2` — "знав", Enter — підтвердити вердикт перевірки (якщо друкував відповідь); Escape — закрити сесію, Enter на фінальному екрані — теж. Підказка клавіш показується внизу картки лише на пристроях з мишею/клавіатурою (`hover: hover`)
- **💬 Чат з репетитором (Claude)** — вбудований AI-репетитор, який бачить твій словник і SRS-прогрес: розмовляє грузинською з підлаштуванням під рівень, виправляє помилки з поясненням правил, будує вправи на твоїх словах. Історія розмови зберігається в базі, відповіді стрімляться. Працює лише онлайн і потребує API-ключ (див. розділ "Чат: налаштування")
- **Додавання слів прямо з чату** — якщо попросиш "додай це слово" або сам погодишся зберегти нове слово з розмови, репетитор додає його в словник (tool use, `add_word`: переклад, за потреби приклад і теги). Клієнт після кожної відповіді чату підтягує свіжий стан словника, тож нове слово одразу з'являється в списку й повтореннях без перезавантаження сторінки
- **📖 Нотатки з граматики** — репетитор сам створює окрему нотатку (`save_grammar_note`: назва + повне пояснення), коли пояснює правило, яке варто зберегти для повторного звернення (особливо якщо явно попросиш "збережи це правило", або тема повторюється); перед створенням перевіряє список наявних, щоб не дублювати тему, і може підтягнути повний текст наявної нотатки (`get_grammar_note`), якщо треба процитувати. У відповіді така нотатка позначається як `[[note:ID|Назва]]` — застосунок показує це клікабельним жетоном "📖 Назва", тап відкриває окреме вікно "Нотатки" з підсвіченою потрібною карткою. У вікні нотаток можна їх переглядати і видаляти (🗑). Токенова вартість: у системний промпт додаються лише **назви** нотаток (не повний текст) — по кілька десятків токенів на нотатку, з кешуванням промпта це не змінює оцінку вартості чату
- **Статистика** (кнопка "📊" в шапці) — слів усього і додано за 7 днів; картки за стадіями (нові / вивчаються (рівень 1–3) / закріплені (4+)); повторено сьогодні, серія днів поспіль 🔥, повторення за останні 7 днів. Журнал повторень по днях зберігається лише локально на пристрої (localStorage `reviewLog`, не синхронізується) — серія/графік рахуються по тому пристрою, де повторюєш
- **Офлайн-режим**: слова зберігаються також у localStorage телефону; можна відкривати аплікацію, шукати і додавати слова без з'єднання з сервером. Нові слова позначаються "⏳ не синхронізовано" і автоматично відправляються на сервер, щойно він доступний (при відкритті сторінки, поверненні мережі або по кнопці "⟳ синхр.")
- Дедуплікація при синхронізації по UUID — слово не задублюється, скільки б разів sync не повторився
- **Повторення з інтервалами (SRS)** — кнопка "🎓 Повторення" відкриває сесію карток: показується слово, згадуєш переклад, тапаєш "Показати відповідь" і оцінюєш себе ("знав"/"не знав"). Кожне слово тренується в обох напрямках (ქართული→переклад і навпаки) з незалежним прогресом. Правильна відповідь збільшує інтервал до наступного показу (1 → 3 → 7 → 14 → 30 → 60 → 120 днів), неправильна скидає на початок і повертає картку ще в цій же сесії. Нових карток максимум 10 за сесію, щоб не завалювало. Працює офлайн; прогрес синхронізується з сервером (конфлікти вирішуються за принципом "новіша оцінка виграє")
- Поле "грузинське слово" має `lang="ka"`, поле "переклад" — `lang="uk"` (правильна семантична розмітка — впливає на вимову скрін-рідерів і мову автокоректу). **Не перемикає системну клавіатуру автоматично** — перевірено на практиці: ні iOS Safari, ні мобільний Chrome не підтримують реальне перемикання розкладки клавіатури через веб-сторінку (на відміну від нативних застосунків типу Google Translate). Клавіатуру між грузинським і українським полем доводиться перемикати вручну — свідомий компроміс, транслітерацію latin→ქართული не впроваджували

## Офлайн на телефоні: разове налаштування

Service worker (механізм офлайн-кешу) браузери дозволяють лише на "безпечних" адресах — https або localhost. `http://192.168.x.x` до них не належить, тому без HTTPS аплікація працює тільки онлайн.

**HTTPS налаштовано через [mkcert](https://github.com/FiloSottile/mkcert)** — локальний центр сертифікації. Все, крім однієї дії на телефоні, автоматизовано:

- При кожному старті `app.py` сам перевіряє, чи є сертифікат для поточної IP-адреси комп'ютера, і за потреби перегенеровує його викликом mkcert (`ensure_cert()` в `app.py`). Сертифікати лежать у `certs/` (не в git — там приватний ключ).
- Якщо на комп'ютері немає mkcert або він вимкнений — сервер тихо падає назад на `http://` (офлайн на iPhone тоді не працюватиме, але онлайн-режим лишається без змін).

**Розово на iPhone (Safari), поки є інтернет:**

1. Відкрий адресу сервера з консолі, напр. `https://192.168.100.113:5000/install-cert` — Safari завантажить профіль
2. Settings → General → VPN & Device Management → відкрий завантажений профіль → Install
3. Settings → General → About → Certificate Trust Settings → увімкни повну довіру для "mkcert ..."
4. Онови/переустанови ярлик "Add to Home Screen" на головній сторінці аплікації

Після цього кожен наступний запуск сервера з тим самим mkcert CA (навіть якщо сертифікат перегенерувався під нову IP) телефон довірятиме автоматично — крок 1–3 повторювати не треба, якщо тільки не перевстановиш mkcert CA на комп'ютері заново.

**Якщо теж користуєшся Android:** той самий трюк з `chrome://flags` більше не потрібен, але CA треба довірити і там (окремо від iPhone і від комп'ютера) — зайди на `/install-cert` у Chrome, файл `.pem` збережеться, потім Settings → Security → Encryption & credentials → Install a certificate → CA certificate. Android покаже попередження "мережу може відстежуватись" — це очікувано для будь-якого власноруч встановленого CA, за одноразовий офлайн-доступ на своєму ж пристрої це прийнятний компроміс.

## Архітектура синхронізації

- Кожне слово має UUID, згенерований на пристрої при створенні
- Телефон зберігає повну копію словника в localStorage і рендерить список з неї
- `POST /api/sync` приймає несинхронізовані слова, вставляє нові (`INSERT OR IGNORE` по UUID), повертає повний список — телефон замінює ним локальну копію
- Синхронізація запускається: при завантаженні сторінки, після додавання слова, при події `online`, по кнопці "⟳ синхр."
- Редагування можливе лише онлайн; видалення несинхронізованого слова — локально, синхронізованого — лише онлайн
- Service worker (`static/sw.js`): стратегія network-first — онлайн завжди свіжа сторінка з сервера (без проблеми "застряглого" кешу), офлайн — остання збережена копія. При зміні логіки кешування піднімати версію в константі `CACHE`
- Мережеві запити (в service worker і в клієнтському `fetch` для sync/delete) мають таймаут 4с (`NETWORK_TIMEOUT_MS`). Без цього при вимкненому Wi-Fi (не просто відсутньому інтернеті) `fetch()` може висіти 20-30с, поки ОС не здасться достукатись до локальної IP, перш ніж впасти в кеш/показати офлайн-статус

## Чат: налаштування (разово)

1. Зареєструйся на https://console.anthropic.com, поповни баланс (вистачить $5) і створи API-ключ
2. Скопіюй `.env.example` як `.env` (у тій же папці, що `app.py`) і встав ключ у `ANTHROPIC_API_KEY=...`
3. Перезапусти сервер

Технічні деталі:

- Модель: `claude-sonnet-5` (константа `CHAT_MODEL` в `app.py`; можна замінити на `claude-opus-4-8` для глибших пояснень — дорожче)
- Системний промпт збирається з бази при кожному повідомленні: всі слова з SRS-прогресом + останні додані нетреновані до межі `CHAT_VOCAB_LIMIT` (300) + статистика; тож при великому словнику (2000+ слів) контекст не роздувається. Системний промпт кешується (prompt caching) — повторні повідомлення в межах ~5 хв платять ~10% за цю частину
- Моделі відправляються останні `CHAT_HISTORY_LIMIT` (30) повідомлень історії
- Відповідь стрімиться (`text/plain` чанки); повна відповідь зберігається в таблицю `chat_messages` після завершення. Помилки API (немає балансу, немає інтернету) показуються в чаті, але в історію не пишуться
- Орієнтовна вартість на Sonnet 5: ~$0.01-0.03 за повідомлення без кешу, з кешем — менше цента; при щоденному використанні — кілька доларів на місяць
- `.env` доданий у `.gitignore` (там секретний ключ); читається з `utf-8-sig`, бо Блокнот/PowerShell пишуть BOM
- Приватність: разом із повідомленнями до Anthropic API відправляється вміст словника (слова, переклади, приклади)

## Структура

- `app.py` — сервер і вся логіка (SQLite, роути, API синхронізації, HTTPS-сертифікати)
- `templates/index.html` — головна сторінка: клієнтський рендеринг з localStorage, форма додавання, пошук, синхронізація
- `templates/edit.html` — сторінка редагування слова (лише онлайн)
- `static/sw.js` — service worker для офлайн-відкриття
- `static/manifest.json`, `static/icon-*.png` — PWA-манифест та іконки для "Add to Home Screen"
- `dictionary.db` — SQLite база, створюється автоматично при першому запуску, **не в git**
- `certs/` — HTTPS-сертифікат (`cert.pem`, `key.pem`) та відмітка IP, для якого він згенерований (`meta.txt`); генерується автоматично, **не в git**
- `.env` — секрети (`ANTHROPIC_API_KEY`), **не в git**; зразок — `.env.example`
- `start.bat` — запуск подвійним кліком на Windows
- `tests/` — серверні тести (`test_*.py`) і клієнтські (`js/test_*.js`); `run_all.py` — раннер, `_client.py`/`js/_harness.js` — спільна інфраструктура тестів (не тести самі по собі)

## HTTPS / mkcert

Потрібен, щоб service worker (а отже й офлайн-режим) працював на iPhone — Safari дозволяє SW лише в "безпечному контексті" (https), без винятків для локальної мережі на відміну від десктопного Chrome.

- Встановлено через `winget install FiloSottile.mkcert`, довірений локальний CA створений командою `mkcert -install` (одноразово на цьому комп'ютері)
- `app.py` при кожному старті сам генерує/перегенеровує лист-сертифікат для поточної IP-адреси через `ensure_cert()` — ручного втручання не треба, навіть коли роутер видає новий IP
- `/install-cert` віддає лише публічний `rootCA.pem` (не приватний ключ) — безпечно віддавати будь-якому пристрою у своїй мережі, з ним не можна підписувати нові сертифікати
- Якщо mkcert не знайдено на машині — сервер автоматично падає назад на звичайний `http://` (без крашу), офлайн на iPhone просто не працюватиме

## База даних

Таблиця `words`: `id, uuid, georgian, translation, example, tags, created_at`. Таблиця `reviews` (SRS-прогрес): `word_uuid, direction ('ka2uk'|'uk2ka'), level, due_at, reviewed_at`, первинний ключ `(word_uuid, direction)`; видалення слова каскадно чистить його оцінки. Таблиця `chat_messages` (історія чату): `id, role ('user'|'assistant'), content, created_at`. Таблиця `grammar_notes`: `id, title, content, created_at`. Схема створюється автоматично, прості міграції — в `init_db()` в `app.py` (там же бекфіл uuid для старих записів).
